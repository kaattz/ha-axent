## 11. 座圈人体感应（Occupancy Detection）

该设备通过 **事件型状态上报** 来表示“是否有人坐下”，而不是通过单一状态位。

### 状态事件

### 坐下（有人）
```text
02-9F-30-12-00-12-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-00-00-00-00-00-00-00-18-D7-0B-04
```

### 起身（无人）
```text
02-9F-30-13-00-13-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-00-00-00-00-00-00-00-18-D7-0B-04
```

---

## 说明

- `30-12` 表示：检测到有人坐下（Occupied）
- `30-13` 表示：检测到无人（Unoccupied）
- 该状态通过 **Notify（FFF1）上报**
- 属于 **事件驱动模型**，而不是持续状态字段

---

### 协议特点

不同于常见设备：

❌ 单个状态包 + bit 位变化  
✅ 不同事件发送不同命令包  

即：

```text
坐下 → 发送 30-12
起身 → 发送 30-13
```

---

### Home Assistant 建模建议

#### binary_sensor

```yaml
binary_sensor:
  - name: toilet_occupied
```

#### 状态逻辑

```text
收到 30-12 → ON（有人）
收到 30-13 → OFF（无人）
```

### 应用场景

- 自动冲水（离开触发）
- 夜灯联动
- 清洗权限控制
- 自动关盖逻辑

---

### 结论

该设备的“是否坐下”状态：

- 通过独立事件包上报
- 不依赖 bit 位
- 是整个自动化逻辑的核心输入信号

# 二、结构分析

## 控制帧结构
02-0E-30-XX-YY-ZZ

含义：
- XX：功能号
- YY：参数 / 模式
- ZZ：校验前字节（通常 XX+YY）

---

## 扩展帧结构
02-9F-30-XX-...

用于：
- 状态上报
- 传感器数据

# 三、清洗系统结构（核心结论）

## 原始观察

- 臀洗、妇洗是独立命令（30-41 / 30-42）
- 声波清洗（30-15）在运行中可切换
- 切换后水流立即变化

---

## 推导模型

清洗 = 动作 + 参数 + 模式

### 动作
- 臀洗
- 妇洗

### 参数
- 水量
- 位置
- 水温

### 模式
- 1D / 2D / 3D（喷射算法）

---

## 关键结论

声波清洗 ≠ 独立功能  
声波清洗 = 喷嘴水流控制算法
