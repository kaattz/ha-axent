### 行为特征
- 可在臀洗/妇洗运行中实时切换

# 七、清洗系统关系（关键）

```text
清洗 = 动作 + 参数 + 模式
```

### 动作
- 臀洗（`30-41`）
- 妇洗（`30-42`）

### 参数
- 水量（`30-46`）
- 位置（`30-36`）
- 水温（`30-16`）

### 模式
- 声波清洗（`30-15`）

## 关键结论
> 声波清洗 ≠ 独立功能  
> 声波清洗 = 喷嘴水流控制算法 / 喷射模式

更准确地说：
- 1D：基础/标准喷射
- 2D：摆动或交替喷射
- 3D：更复杂的脉冲 + 摆动喷射

这是根据实际测试“切换时出水感觉实时变化”得出的结论。


## 状态模型
```text
坐下 → 30-12
离开 → 30-13
```

### 关键结论
- 这是“事件模型”
- 不是通过一个状态位切换 0/1
- 而是不同状态发送不同事件包

### Home Assistant 建模建议
- `binary_sensor.toilet_occupied`
- 收到 `30-12` → `on`
- 收到 `30-13` → `off`

### 应用场景
- 离开触发自动冲水
- 夜灯联动
- 清洗权限控制
- 自动关盖逻辑

# 十四、结构分析

## 控制帧结构
```text
02-0E-30-XX-YY-ZZ
```

含义：
- `XX`：功能号
- `YY`：参数 / 模式 / 档位
- `ZZ`：校验前置字节（通常 = `XX + YY`）

## 扩展帧结构
```text
02-9F-30-XX-...
```

用于：
- 状态上报
- 传感器数据
- 高级参数块写入

# 十五、协议总结

## 协议特性

- 无加密
- 结构统一
- 支持实时控制
- 支持事件上报
- 存在控制帧与扩展帧两套结构

## 架构模型
```text
HA → BLE Write(FFF2) → 马桶
马桶 → Notify(FFF1) → HA
```

## 最终结论

> 该协议属于一个完整的“实时控制系统协议”

支持：
- 全功能控制
- 实时参数调整
- 状态同步
- 替代官方 APP
